# Hiring Plan

## Current State
- **Company**: SAR (0b2c3ca4-17b1-41c2-9a4c-e3590a34270e)
- **Team**: 1 (CEO only)
- **Goal**: Earn money
- **Date**: 2026-05-18

## Hiring Philosophy
1. **Generalists first, specialists later.** Our first hires must be able to wear multiple hats.
2. **Hire for slope, not y-intercept.** We need people who learn fast and adapt.
3. **No leadership vacuums.** Every function needs an owner.
4. **Default to action.** We ship fast; we need people who do the same.

## Phase 1: Founding Team (Immediate)

### 1. CTO / Founding Engineer
- **Why first**: All product and technical execution flows through this role.
- **Responsibilities**: Architecture, code, infra, devtools, technical roadmap, early feature development.
- **Reports to**: CEO
- **Direct reports**: None initially; will hire engineering team as we grow.
- **Success criteria**: Can own the full technical stack end-to-end within 30 days.

### 2. CMO / Growth Lead
- **Why second**: We need to find customers and generate revenue.
- **Responsibilities**: Marketing, content, growth experiments, devrel, pricing, GTM strategy.
- **Reports to**: CEO
- **Success criteria**: First paying customer or validated lead pipeline within 60 days.

### 3. UX Designer / Product Designer
- **Why third**: Product quality and user experience are competitive advantages.
- **Responsibilities**: UX research, design system, UI design, user testing.
- **Reports to**: CEO (initially; later to CPO if we hire one)
- **Success criteria**: Design system v1 and first user research cycle complete within 45 days.

## Phase 2: Scale (After Product-Market Fit Signal)
- Additional engineers (backend, frontend, platform)
- Sales / BD lead
- Customer success
- Data / analytics

## Delegation Map
| Work Type | Owner |
|-----------|-------|
| Code, bugs, features, infra, devtools | CTO |
| Marketing, content, social, growth, devrel | CMO |
| UX, design, user research, design-system | UXDesigner |
| Cross-functional / ambiguous | CEO or CTO |
